# Development Roadmap

Development proceeds phase by phase. Completed phases are grouped into public releases.

| Phase | Development unit | Main output |
|---:|---|---|
| 0 | Product thesis and evaluation contract | Executable specifications, seed incidents, safety and acceptance gates |
| 1 | Synthetic commerce environment | Reproducible customers, orders, payments, inventory, fulfilment, and operations data |
| 2 | Analytics and metric layer | Funnels, cohorts, contribution analysis, metric definitions, data quality |
| 3 | Advanced anomaly detection | Seasonal baselines, robust statistics, change points, sequential tests, FDR control |
| 4 | Churn and customer impact | Survival models, calibration, incident exposure, causal impact, revenue risk |
| 5 | Evidence and lineage | Deployments, logs, pipeline records, graph relationships, incident retrieval |
| 6 | Safe tool gateway | Read-only SQL, policies, MCP tools, timeouts, audit logs |
| 7 | Probabilistic agentic investigation | State machine, tool selection, hypotheses, contradiction search, probability ranking |
| 8 | Approval, remediation, recovery | Human approval, reversible actions, guardrails, reopening |
| 9 | Evaluation and adversarial testing | Hidden benchmark, baselines, ablations, regression and security tests |
| 10 | TUI and web product | Developer TUI and public live investigation dashboard |
| 11 | Production and portfolio packaging | Docker, CI, hosted demo, observability, video, technical article |

## Release grouping

- **Release 1 - Quantitative Foundation:** Phases 0-4
- **Release 2 - Agentic Investigation:** Phases 5-7
- **Release 3 - Governed Operations Product:** Phases 8-10
- **Release 4 - Portfolio Proof:** Phase 11 and final benchmark publication

Every phase requires working code, tests, updated status, a pull request, and independent review before the next phase begins.
