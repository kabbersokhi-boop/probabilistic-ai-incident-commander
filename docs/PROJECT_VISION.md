# Project Vision

## Product

**Probabilistic AI Incident Commander** is a portfolio-grade, evidence-grounded agentic system for synthetic commerce operations.

It will:

1. detect statistically meaningful anomalies,
2. identify affected cohorts and business impact,
3. plan an investigation,
4. query controlled tools across SQL, logs, deployments, lineage, configurations, and incident history,
5. generate and falsify competing hypotheses,
6. rank root causes using externally calculated probabilities,
7. estimate immediate revenue and longer-term churn impact,
8. request human approval for sensitive remediation,
9. simulate the approved action,
10. verify sustained recovery with guardrail metrics,
11. produce an evidence-backed incident report,
12. score the complete run against hidden ground truth.

## Portfolio thesis

The project must demonstrate one coherent combination of:

- advanced data analysis and SQL,
- statistical programming and uncertainty,
- probability calibration,
- churn and survival analysis,
- causal reasoning,
- data and lineage engineering,
- controlled agentic AI,
- security and human approval,
- evaluations and failure analysis,
- backend, frontend, testing, and deployment engineering.

## Differentiation standard

The system is not successful merely because one demonstration works. It must publish reproducible benchmark results, baselines, ablations, safety tests, and known failure modes.

## Non-goals

- Real production access during the portfolio build
- An unrestricted autonomous operator
- A multi-agent swarm without measured need
- A collection of disconnected notebooks
- Infrastructure added only for visual complexity
